Code Dialog Quick Help
What is Code Dialog?
The code dialog accepts the key for encryption or decryption process. Here you can enter the code, set the hint and select the alogrithm for the process. Each element of this window are discussed below
The code
The code for the process is used for the encryption and decryption. The code should be a numberical value or a mix of numbers and characters. In a mixed mode code the digits will be taken from left to right and they will be converted into a number and the program will convert the characters into corresponding values and add them to the value obtained. In any case the code should be numerical value above 10000 and below 1000000000000000000000000000000000000000000000000000000 ( ie 50 digits). for eg if the code entered is m8jd9kd90493033uik23dde78 Here the numerical value of the code will be 89904930332378 and the remaining characters will be converted into digit values and will be added to the numerical value.
Setting the hint
You can set a hint for the code during encryption. For setting a hint click on the 'Set hint for code' button and enter the hint in the input dialog. Now during decryption the there will be a button captioned 'show hint' which will show the hint for the code you entered during encryption .
Defining the algorithms
in the version 1.0, you just need to select an algorithm from the list. But from version 2.0 onwards, the program will be using the Multilevel encryption technology. So to define the algorithms to use, click on the 'Change algorithm hierarchy' button, then a new window will pop up. To know more about Multilevel encryption refer to the quick help provided on that window. The selected algorithms hierarchy will be displayed on the Code Dialog when done.
A word of caution. The algorithms play with numbers, so many codes may generate many sequence and the wrong code entered for decryption may actually decrypt the file to a junk value. So keep a backup copy of encrypted file.But it is guaranteed that only the correct code will give you the correct decryption . Even a change in last digit by 1 value will give the wrong output. So do have full faith in the program.
Save Encryption Settings Option
This is a new option added in version 2.0. In version 1.0, when an encrypted file is added all the details of encryption such as algorithm used, whether equation used , hint etc (code and equation are not stored any where) are also stored along with encrypted data. It is a great assistance during decryption. But some times discarding the storing of these settings can be a added security feature. Also in Multilevel encryption the order of algorithms is very important, So if they are not saved, the algorithms need to be specified in correct order and provides more security. To enable this feature this check box is used. If the checkbox is checked, then the settings will be stored else the settings will be discarded and will not be stored anywhere.
Though discarding the settings provide more security, the user should provide correct order of algorithms, an equation if it is used etc. A small mistake by the authorized user can yield wrong output rendering the file useless.
NOTE: Even if the checkbox is not checked , you can still set a hint for the process.
 
Equation
It is an added security feature for the code. The equation allows you to specify an equation and the code will be modified according the equation. It is like using dual key for encryption and the file will get decrypted only if both the code and equation is correct . The Equation is discussed in detail in the equation dialog that is presented to you when you click the ' Use Equation ' button.
Encryption
During the encryption process the characters will be encrypted to numerical values and the speed of encryption will be more than 10000 characters per minute ( depends upon the resources available). After the encryption the size of the file becomes four times the actual size, The encrypted files can be only saved with .MENCF extension and they will be restored to the previous extension when they are decrypted.
