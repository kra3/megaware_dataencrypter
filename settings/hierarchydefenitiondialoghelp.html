Multilevel Hierarchy Defenition Dialog Quick Help
 
What is Multilevel Hierarchy Defenition Dialog?
The Multilevel Hierarchy Defenition Dialog is the same Algorithm Selection Dialog used in version 1.0 of encryter. But in version 2.0 we are introducing a new high security technology that is, Multilevel encryption . As the term refers, the data can be encrypted using different algorithms one after other. This series of algorithms to be used are defined using Multilevel Hierarchy Defenition Dialog which is discussed below.
 
How is Multi level encryption secure?
In Multi level encryption , you specify many algorithms as a chain and will be represented in a list. During encryption the data will be first encrypted using the first top level algorithm and the output of this algorithm, ie an encrypted content is again encrypted using the second algorithm in the list and so on. It makes the code unbreakable .
But one disadvantage is that, as the number of algorithms increase so does the size of the file and the time taken for encryption or decryption process.But this can be overcome by wise usage of algorithms in hierarchy.
 
How to define the algorithms Hierarchy
In the dialog, there will be two list boxes, the list in the left side contains the available algorithms. The right hand side list contains the hierarchy of algorithms. You can add the algorithms from the left list to the right one. The same algorithm can be added to the hierarchy many times. After adding all the algorithms and ordering them, click the 'Done' button and the hierarchy will be displayed in Code Dialog
 
Adding algorithms to Hierarchy
To add an algorithm to the hierarchy list, just click the required algorithm on the left side and then click the button ' Add to list'. The algorithm will be added to the hierarchy list. You can add the same algorithm as many times as needed and you can also use a single algorithm.
 
Setting the order of algorithms in the hierarchy list

Usually the algorithms added to the list are appended to the hierarchy list. But as said before, the algorithms are used from top to bottom and efficient use of algorithms can save time and provide maximum security. So if you have added the required algorithms to the list you can change their order by clicking the 'MOVE UP' and 'MOVE DOWN' buttons. Select an algorithm in the hierarchy list and then click 'MOVE UP'. It will move the selected algorithm one level up and same procedure with 'MOVE DOWN' will move the selected algorithm one level down the hierarchy.According to me an efficient pair is ALGORITHM META CBE --> ALGORITHM SHUFFLE.ie , on the top of hierarchy ALGORITHM META CBE and next ALGORITHM SHUFFLE.

 
Removing an algorithm from the hierarchy list
If you want to remove an item (algorithm) from the hierarchy, then select the algorithm from the list and click the ' Remove from list' button. This will remove the selected algorithm from the list. Note: You can remove only algorithms in the hierarchy list.
 
Setting an algorithm as default
The default algorithm is the algorithm that will be selected automatically in the Code Dialog for a new Encryption. It will be also used for decryption if the settings data for decryption is not found. You can set any algorithm as default. To set an algorithm as default, select the algorithm from the Available Algorithms list and then click the 'Set as default Algorithm' button. This will pop up a message showing successful change of setting if the selected algorithm is set as default.
 
Getting the description about an Algorithm
In Multi level encryption, it is important that , the user must know the advantages and disadvanges of an algorithm for high security and low encryption time. For that each algorithm is provided with a description about its characteristics, advantages, disadvantages etc. To view the description about an algorithm, select the algorithm in the Available Algorithms list and then click the 'Show description of Algorithm'' button. This will open up a new window containing the description of algorithms.
